### Finding 5.3: PostGIS Port Exposed on All Interfaces
- **Severity:** MEDIUM
- **Category:** Security
- **Files Affected:** `docker-compose.yml`
- **Description:** PostGIS is exposed as `"5432:5432"` which binds to 0.0.0.0 — accessible from any network interface. For local dev, should only be accessible on localhost.
- **Evidence:** docker-compose.yml line 22: `- "5432:5432"`
- **Recommendation:** Change to `"127.0.0.1:5432:5432"`
- **Effort:** S | **Risk:** LOW

### Finding 5.4: Docker Socket Mount — Security Risk
- **Severity:** MEDIUM
- **Category:** Security
- **Files Affected:** `docker-compose.yml`
- **Description:** LocalStack mounts `/var/run/docker.sock` inside the container, giving it full Docker daemon access. A compromised LocalStack container could control the host Docker.
- **Evidence:** docker-compose.yml line 58: `- /var/run/docker.sock:/var/run/docker.sock`
- **Recommendation:** Use Docker-out-of-Docker (DooD) alternative or remove Docker socket mount if not needed for LocalStack's S3/Lambda emulation
- **Effort:** S | **Risk:** MED

### Finding 5.8: Security Headers — Comprehensive but CSP Allows unsafe-eval
- **Severity:** MEDIUM
- **Category:** Security
- **Files Affected:** `next.config.ts`
- **Description:** HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy all set correctly. However, CSP includes `unsafe-eval` and `unsafe-inline` for scripts which weakens XSS protection.
- **Evidence:** next.config.ts line 44: `"script-src 'self' 'unsafe-eval' 'unsafe-inline' https://cesium.com"`
- **Recommendation:** CesiumJS requires `unsafe-eval` for its Workers. Document this as a known CSP exception. Try to remove `unsafe-inline` by using nonces.
- **Effort:** M | **Risk:** MED
