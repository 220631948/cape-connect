### Finding 4.2: SECURITY DEFINER RPCs — Tenant Check Present
- **Severity:** MEDIUM
- **Category:** Security
- **Files Affected:** `20260310030000_user_features.sql`, `20260310020000_property_search.sql`
- **Description:** `analyze_area()` and `search_properties()` use SECURITY DEFINER (runs with function owner's privileges). Both correctly enforce `app.current_tenant` internally.
- **Evidence:** analyze_area line 58: `AND p2.tenant_id = current_setting('app.current_tenant', TRUE)::uuid`; search_properties line 39: `AND p.tenant_id = current_setting('app.current_tenant', TRUE)::uuid`
- **Recommendation:** Add explicit search_path setting to prevent hijacking: `SET search_path = public` in function definition. Also add `ROWS` estimate for query planner.
- **Effort:** S | **Risk:** LOW

### Finding 4.3: Missing ST_IsValid Constraints
- **Severity:** MEDIUM
- **Category:** Architecture
- **Files Affected:** Initial schema (properties, zones tables)
- **Description:** CLAUDE.md §3 mentions `CHECK (ST_IsValid(geom))` constraints but none are present in the migrations. Invalid geometries could be inserted and cause query failures.
- **Evidence:** `grep -r "ST_IsValid" supabase/migrations/` → no results
- **Recommendation:** Add `ALTER TABLE properties ADD CONSTRAINT properties_valid_geom CHECK (ST_IsValid(geometry));` for properties and zones tables
- **Effort:** S | **Risk:** LOW
- **Acceptance Criteria:** Invalid geometries rejected on insert

### Finding 4.4: PostgreSQL Version Mismatch (Docker 17 vs Supabase 15)
- **Severity:** LOW
- **Category:** Architecture
- **Files Affected:** `docker-compose.yml`, all migrations
- **Description:** Local dev uses `kartoza/postgis:17-3.5` (PostgreSQL 17) but Supabase hosted runs PostgreSQL 15. The composite spatial indexes migration uses `INCLUDE` columns in GiST indexes which require PostgreSQL 14+ (both versions support this).
- **Evidence:** docker-compose.yml line 6: `image: kartoza/postgis:17-3.5`; CLAUDE.md §2: "PostgreSQL 15"
- **Recommendation:** Monitor for PostgreSQL 17-only features used in migrations. Consider using `kartoza/postgis:15-3.4` for closer parity.
- **Effort:** S | **Risk:** LOW
