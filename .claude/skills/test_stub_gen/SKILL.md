## Purpose

Bootstrap test coverage for untested components by generating opinionated Vitest stubs.
Stubs include the required mocks for MapLibre GL JS and Supabase, extract component props
from TypeScript interface definitions, and scaffold one describe/it block per exported
function or component. Developer must implement assertions before merge.

## Trigger Conditions

- `/coverage-report` identifies zero-coverage components
- M17-ANALYSIS-AGENT creates new components
- TEST-COVERAGE-AGENT activation
- `test_stub_gen` explicitly invoked with a component path

## Procedure

1. **Read target component file** at provided path.
   Extract:
   - Component name (default export or named export)
   - Props interface (TypeScript `interface Props` or `type Props`)
   - `useEffect` dependencies (implies async behavior to test)
   - API calls: `fetch(`, `supabase.from(`, `supabase.rpc(`
   - MapLibre references: `mapRef`, `map.addLayer`, `map.addSource`
   - External hooks: `useAnalysis`, `useTenant`, `useLiveData`

2. **Determine mock requirements:**
   - MapLibre reference found → include `vi.mock('maplibre-gl', ...)`
   - Supabase call found → include `vi.mock('@/lib/supabase', ...)`
   - API fetch found → include `global.fetch = vi.fn()`
   - Custom hook found → include `vi.mock('@/hooks/<hook>', ...)`

3. **Generate stub file structure:**
   ```typescript
   // STUB: implement assertions before merge
   // Generated by test_stub_gen on <date>
   import { render, screen } from '@testing-library/react';
   import { describe, it, expect, vi } from 'vitest';
   import <ComponentName> from '@/components/<path>';

   // Required mocks
   vi.mock('maplibre-gl', () => ({ Map: vi.fn(() => ({ ... })) }));
   vi.mock('@/lib/supabase', () => ({ supabase: { from: vi.fn() } }));

   describe('<ComponentName>', () => {
     it('renders without crashing', () => {
       // STUB: add render assertions
       render(<ComponentName {...defaultProps} />);
       expect(true).toBe(true); // placeholder
     });

     it('calls data fetch on mount', async () => {
       // STUB: assert fetch called with correct args
     });

     // Additional it() blocks for each prop/behavior extracted
   });
   ```

4. **Write stub to:** `src/__tests__/<ComponentName>.test.tsx`
   - Skip if a test file already exists with real assertions (non-STUB content)
   - Overwrite only if existing file has `// STUB` marker and no custom assertions

5. **Report:** List generated stubs + components that already have tests.

## When NOT to Use

- For utility functions (use Vitest directly without this skill)
- When a component already has comprehensive tests (check for `// STUB` marker first)
- For Playwright E2E tests (this skill generates unit stubs only)
- On files in `src/test/` or `src/__tests__/` themselves
