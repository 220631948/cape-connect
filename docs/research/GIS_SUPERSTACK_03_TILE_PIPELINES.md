# GIS_SUPERSTACK_03_TILE_PIPELINES

This document outlines the vector and map tile pipeline strategy for the CapeTown GIS Hub (`capegis`), covering generation, serving, and offline caching patterns.

## Domain Goal

Design an end-to-end tile pipeline from PostGIS data to rendered client tiles, including serverless and offline/PWA strategies.

## 1. Generation Pipeline (Tippecanoe)

**Status**: ✓ VERIFIED [Source: Felt] | **Recommended**: P0

For large datasets (e.g., 830k parcels), generating MVT tiles in real-time is too heavy for the server. Pre-generating tiles via Tippecanoe is mandatory for static or slowly-changing layers.

### Canonical Workflow:

1.  **Extract**: `ogr2ogr -f GeoJSON parcels.geojson PG:dbname=capegis parcels`
2.  **Generate**: `tippecanoe -o parcels.pmtiles -zg --drop-densest-as-needed -l parcels --clip-bounding-box=18.0,-34.5,19.5,-33.0 parcels.geojson`
3.  **Deploy**: Upload `parcels.pmtiles` to Supabase Storage or S3.

---

## 2. Serving Strategy: PMTiles v3 (Serverless)

**Status**: ✓ VERIFIED [Source: https://protomaps.com/] | **Recommended**: P1 (NEW)

PMTiles v3 allows serving map tiles via standard HTTP range requests from any object storage (S3, Supabase Storage, Cloudflare R2).

### Why PMTiles beats MBTiles:

- No active backend process required.
- Infinite scalability via global CDN.
- Cost-effective for 1M+ requests.

### Implementation Pattern (Next.js/Cloudflare):

Use a Cloudflare Worker or a light API route in Next.js to proxy range requests to the `.pmtiles` file.

---

## 3. Dynamic Vector Tiles: Martin (Rust)

**Status**: ✓ VERIFIED [Source: https://github.com/maplibre/martin] | **Recommended**: P0 (Dynamic Only)

Retain Martin for layers that change frequently (e.g., user-drawn features, real-time overlays).

### Optimization Rule:

Always append `?optimize=true` to Martin source URLs to enable server-side pruning of small geometries at low zoom levels.

---

## 4. Offline / PWA Strategy (Serwist)

**Status**: ✓ VERIFIED | **Recommended**: P0

For the Western Cape area of interest (AOI), the PWA should pre-cache basemap tiles at zoom levels 8-12 and cache specific `.pmtiles` archives for offline parcel inspection.

### Service Worker Configuration:

Configure Serwist to intercept map requests and serve from the local `IndexedDB` or `CacheStorage`.

---

## Recommended Implementation Path

- **Static Layers (Parcels, Zoning, Suburbs)**: Move to **PMTiles v3** hosted on Supabase Storage.
- **Dynamic Layers (User Analysis, Real-time)**: Retain **Martin MVT**.
- **Basemaps**: Continue using **Protomaps** or **Stadia Maps** via PMTiles for offline resilience.
