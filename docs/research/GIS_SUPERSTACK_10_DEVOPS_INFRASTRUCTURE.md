# GIS_SUPERSTACK_10_DEVOPS_INFRASTRUCTURE

This document details the CI/CD, observability, and infrastructure tooling for the CapeTown GIS Hub (`capegis`), focusing on the operational complexity of spatial databases and tile pipelines.

## Domain Goal

Design infrastructure as code (IaC) and observability that scales with spatial data volume and complex rendering pipelines.

## 1. CI/CD for GIS Data

**Status**: ✓ VERIFIED | **Recommended**: P0

Automated validation of geospatial files (GeoJSON, Parquet) before deployment is critical to prevent "empty map" failures.

### GitHub Actions Pipeline:

1.  **Validate**: `npx -y geojson-lint public/mock/*.geojson`.
2.  **Generate**: Convert new datasets to PMTiles using Tippecanoe in CI.
3.  **Upload**: Push generated PMTiles to object storage (Supabase Storage).
4.  **Audit**: Run `/badge-check` and `/validate-spatial` via CLI agents.

---

## 2. Observability: Datadog Geospatial

**Status**: ✓ VERIFIED [Source: Datadog 2026 Update] | **Recommended**: P2

Use Datadog's new geospatial metrics to visualize request heatmaps and identify tile server hotspots or latency issues in specific regions (e.g., Table Mountain shadows).

### Key Metrics:

- Tile render time (Martin).
- Spatial join latency (PostGIS/DuckDB).
- Cache hit ratio (Serwist).

---

## 3. Scalable Spatial Database Hosting

**Status**: ✓ VERIFIED | **Recommended**: P0

Supabase (PostgreSQL 17 + PostGIS 3.6.2) is the primary hosting platform. **pgBouncer** must be configured to handle the surge of connections from the Next.js serverless functions.

---

## 4. Serverless Geospatial Delivery (S3 + PMTiles)

**Status**: ✓ VERIFIED | **Recommended**: P1 (NEW)

The shift to serverless tiling (PMTiles v3) simplifies DevOps by removing the need for dedicated Martin or Tegola servers for static layers.

### Infrastructure Pattern:

1.  **S3 / R2 Bucket**: Stores all `.pmtiles` files.
2.  **Cloudflare Workers**: Handles HTTP range requests and adds a simple caching layer.
3.  **Next.js Client**: MapLibre fetches tiles directly from the Edge.

---

## Recommended Implementation Path

1.  **CI PMTiles Pipeline**: Implement a GitHub Action to convert and upload cadastral updates as PMTiles.
2.  **pgBouncer Audit**: Verify connection pool settings in the Supabase dashboard.
3.  **Datadog Integration**: Wire basic app performance metrics (Next.js) before adding spatial-specific ones.
