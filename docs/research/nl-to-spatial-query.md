## 1. Overview
Research into Autonomous GIS agents demonstrates that translating natural language (NL) into spatial queries is a maturing field, moving from simple SQL generation to multi-step reasoning agents.

## 2. Key Systems & Research
- **GIS Copilot (Akinboyewa et al., 2025):** An autonomous agent plugin for QGIS. It leverages LLMs to generate spatial workflows and Python code (GeoPandas) from NL.
- **MapAgent (Hasan et al., 2025):** A hierarchical agent architecture split into a "Planner" and an "Executor." This prevents the LLM from getting lost in complex geoprocessing chains.
- **Spatial-Agent (Bao et al., 2026):** Utilizes **GeoFlow Graphs** to decompose spatial concepts into deterministic execution paths, reducing hallucinations.

## 3. Technical Patterns
- **Spatial Intent Parsing:** The LLM must first identify if a query requires **Retrieval** (e.g., "Find the nearest hospital") or **Processing** (e.g., "Calculate the slope of this terrain").
- **Geoprocessing Tool Registries:** Agents are most effective when they have a strict "toolbox" of PostGIS or Turf.js functions they are allowed to call, rather than writing arbitrary code.
- **CRS Disambiguation:** High-quality agents automatically check the `.prj` or metadata of a layer to ensure coordinate system compatibility before performing intersections.

## 4. The "Archer" Architecture (2026 Standard)
Using **LangGraph** to build a robust **Plan-Verify-Execute** loop for spatial reasoning:
1.  **Intent Parser:** Detects geospatial intent from natural language.
2.  **Task Planner:** Breaks requests into geoprocessing sub-tasks.
3.  **Tool Registry:** Selects tools from the MCP registry (e.g., `buffer`, `intersect`).
4.  **Verification:** Validates the output geometry before rendering to the 4D WorldView to prevent "hallucinated geometry."

## 5. Competitive Landscape
- **ArcGIS AI Assistants:** Proprietary, tightly integrated with Esri's GeoAI framework. Focused on explainable decisions.
- **Open-Source Copilots:** Rely on LLM coding capabilities (Python/SQL) and semantic search over GIS documentation.
