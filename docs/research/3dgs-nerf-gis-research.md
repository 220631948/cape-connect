## 1. Overview
Research into browser-based 3D reconstruction pipelines reveals a significant shift from volumetric neural representations (NeRF) to point-based radiance fields (3D Gaussian Splatting) for real-time GIS applications.

## 2. Comparative Analysis

## 3. Technical Requirements
- **Hardware:** NVIDIA RTX 3090/4090 (24GB VRAM) is the baseline for efficient training of Splatfacto and Instant-NGP.
- **Datasets:** Requires high-overlap photogrammetry datasets (minimum 60-80% overlap). 3DGS is more robust to sparse views than vanilla NeRF but benefits from COLMAP-derived camera poses.

## 4. CesiumJS Integration
- **KHR_gaussian_splatting:** A Khronos glTF extension (Release Candidate: Feb 2026) now enables standardized storage of Gaussian splats within `.gltf`/`.glb` files.
- **CesiumJS Support:** Native support for Gaussian Splatting was introduced in CesiumJS v1.131 (verified via community issues #12718); full production support for **KHR_gaussian_splatting** is available in **v1.139+** (March 2026).
- **Cesium ion Workflows:** Users can upload `.ply` or `.spz` splat files to Cesium ion, which tiles them into 3D Tiles for efficient streaming.
- **SPZ Format:** A compressed Gaussian splatting format gaining traction for web delivery (UNVERIFIED exact compression ratios compared to Draco).

## 5. Conclusions
For the Cape Town GIS Platform, **3D Gaussian Splatting (3DGS)** is the recommended standard over NeRF. 3DGS's reliance on GPU rasterization rather than volumetric ray-marching allows for high-performance rendering on consumer-grade browsers and mobile devices.
